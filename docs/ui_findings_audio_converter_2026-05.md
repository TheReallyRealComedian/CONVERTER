# UX-Heuristik-Findings: audio_converter (2026-05-03)

**Methodik:** Stufe 2 der Duan-Kaskade (Duan et al., *Heuristic Evaluation with LLMs*, CHI 2024). Heuristisches Review der strukturierten Inventur aus Stufe 1.
**Quelle:** [docs/ui_inventory_audio_converter_2026-05.md](ui_inventory_audio_converter_2026-05.md)
**Heuristiken:** Nielsen H1 (Sichtbarkeit des Systemzustands), H4 (Konsistenz und Standards), H6 (Wiedererkennen statt Erinnern), H9 (Fehlermeldungen / Hilfe bei Fehlern)
**Produkt-Kontext:** Single-User (Oliver), LAN-only, login-protected. Technisch versierter Nutzer; H4 bekommt **erhöhtes Gewicht** wegen *Cross-Feature*-Dimension — F-1 (`document_converter`) hat App-Konventionen etabliert (`showAlert`, `showToast`, echtes Drag-Drop, DE-Microcopy, `formatFileSize`, Save-Btn-Lifecycle, `:focus-visible` auf c-btn, Backend-Whitelist+DE-Fehler), die `audio_converter` nicht befolgt. H1 stark wegen Recording- & Permission-Pfaden; H9 stark wegen Mic-Hardware-/Permission-Failures; H6 erhöht wegen versteckter Audio-States. a11y nicht null gewichtet — Tastaturbedienung wird genutzt.

## Reine Implementierungs-Bugs (kein eigenständiges Heuristik-Finding, separates Ticket-Material)

Diese Befunde sind in den Findings oben **bereits aus UX-Sicht erfasst**, brauchen aber zusätzlich konkrete Code-Fixes. Plus ein paar pure Bugs, die kein Heuristik-Finding ergeben.

- **B1: Drag-Drop-Handler fehlen.** Keine `dragover/dragleave/drop`-Listener in [static/js/audio_converter.js](static/js/audio_converter.js). → **siehe Findings F3, F4.**
- **B2: `.saved`-Klasse wird beim Reset des Save-Buttons nicht entfernt.** Zwei Code-Stellen: [static/js/audio_converter.js:589-590](static/js/audio_converter.js:589) (`#save-transcription-btn`) und [:626-628](static/js/audio_converter.js:626) (`#save-live-btn`). Identisch zum F-1.B2-Bug. → **siehe Findings F9, F10.**
- **B3: `getUserMedia` hat kein `.catch()` für `denied`/`NotAllowedError`/`NotFoundError`.** Der Pfad fällt durch das async-onopen ohne explizite Behandlung. → **siehe Finding F5.**
- **B4: Result-Container Copy-Guard greift nicht beim Placeholder.** `copyToClipboard`-Guard prüft `text.trim()===''`, aber der Placeholder-Text "The transcription will appear here after processing." ist non-empty. Fix: zusätzliche Prüfung gegen einen Sentinel (data-Attribut oder textContent-Vergleich). → **siehe Finding F17.**
- **B5: Live-Transcript `socket.onmessage` überschreibt User-Edits in jedem interim-Frame.** Solange User in der Textarea tippt, sollte der Append-Pfad pausieren oder die Textarea während Recording readonly setzen. → **siehe Finding F16.**
- **B6: Inline `<span>`-Error-Reporting nutzt raw `innerHTML` mit interpoliertem `err.message`.** XSS-Risiko theoretisch (Single-User-LAN: niedrig), aber Pattern sollte so oder so durch `showAlert` ersetzt werden. → **siehe Finding F8.**
- **B7: Generate-Script-Button-Restore hat hartkodiertes SVG-Markup im `finally`-Block** ([static/js/audio_converter.js](static/js/audio_converter.js), `generateScriptBtn.innerHTML = '<svg…>Generate Script…'`). Code-Duplikation zur Template-Definition; bei Template-Änderung wird der Restore stale. **Pure Bug, kein Finding** — nur Maintainability-Risiko. Fix: Initial-innerHTML in einer Konstante oder einem `data-default-html`-Attribut speichern.
- **B8: WS-Auth-Failure-Pfad gibt generische `'Connection failed. Check your network and API configuration.'`-Meldung aus**, unabhängig davon, ob es ein Token-Fail (401), ein Network-Fail (offline) oder ein WS-Handshake-Fail (4xx) war. **Pure Bug** — die Recovery-Anleitung könnte je nach Ursache spezifischer sein. Schließt teilweise an F14 an, aber ist getrennt davon (F14 = Konfigurations-Banner, B8 = Runtime-WS-Failure).
- **B9: `transcribe_file` Backend hat `MAX_FILE_SIZE_MB=500`, aber Frontend hat keinen Pre-Submit-Filesize-Check.** User uploadet 600 MB → wartet auf Backend-Reject. **Pure Bug** im Sinne "fehlende Frontend-Validation". Sollte mit F19 (accept-Whitelist + Backend-Validation) zusammen in Stage 3 angegangen werden.
